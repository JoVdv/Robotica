# De opdrachten

## Portfolio
Waarom een portfolio? 

Het portfolio kan per student de praktische als theoretische kennis aantonen. Een goed portfolio laat niet alleen zien wat er is gedaan, maar ook wat er is geleerd en hoe de kennis is toegepast.

het door de student gemaakte portfolio toont de gemaakte oefeningen en projecten, alsook datgene wat hij/zij heeft geleerd.


Eisen aan het portfolio

    Het portfolio omvat één document met verwijzingen naar de nodige bijlage.
    Let op dat de bijlage kan ge-opend worden door de lector.
    In het portfolio kunnen code-snippets staan, maar nooit meer dan een halve pagina. Grotere stukken code worden bijlagen.
    Informatie groter dan een halve pagina, bij voorkeur als bijlage toevoegen.
    Het portfolio heeft een logische inhoudstafel en een logische structuur.
    Zie de template op Blacboard voor een voorbeeldstructuur.
    Laat het portfolio controleren door de letor voor feedback.  Doe dit voor de deadline!


Inhoud van het portfolio


    Gemaakte oefeningen m.b.t robotica (= programma's op de robot en simulatie)
    = Beschrijving: programma code; uitwerking en resultaat; reflectie/analyse
    Gevolgde cursussen
    = Omschrijving, onderwerpen, geleerde kennis
    Datasheet/fabrikanten info
    TIF (technical information sheet) **
    = Een probleemanalyse en oplossing
    =Specifiek probleem wat je bent tegengekomen met een omschrijving hoe je dit hebt opgelost en linken naar de documenten
    logboek **
    chronologische registratie van de bestede uren en de uitgevoerde taken per sessie
    Technische blog **
     korte artikelen of notities die specifieke technische uitdagingen, aha-momenten of geleerde lessen
    Git-profiel **
    Link naar je git-hub account waar documentatie en code is gedeeld.
    Eigen aanvulling **

** = Extra toevoeging, = geen verplichte inhoud.
